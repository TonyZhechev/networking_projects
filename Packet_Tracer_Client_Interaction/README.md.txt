# 🚀 PACKET TRACER - CLIENT-SERVER INTERACTION 🌐  

## 📌 PROJECT OVERVIEW  
THIS PROJECT DEMONSTRATES HOW A **CLIENT (PC) INTERACTS WITH A SERVER** IN A NETWORK.  
YOU WILL OBSERVE **REQUESTS & RESPONSES**, TEST CONNECTIVITY, AND LEARN HOW DATA FLOWS IN A NETWORK!

## 🛠️ **PROJECT OBJECTIVES**
✔️ **CONNECT DEVICES** – SERVER AND CLIENT PCS CONNECTED TO A SWITCH.  
✔️ **CONFIGURE IP ADDRESSES** – EACH DEVICE GETS A UNIQUE IP ADDRESS.  
✔️ **TEST CONNECTIVITY** – USING `PING` COMMAND TO VERIFY CONNECTIONS.  
✔️ **OBSERVE DATA FLOW** – MONITOR HOW CLIENT REQUESTS & SERVER RESPONSES TRAVEL.

## 🚀 **HOW TO USE THIS PROJECT**  

### **1️⃣ OPEN PACKET TRACER**
- LAUNCH **Cisco Packet Tracer**.  

### **2️⃣ LOAD THE PROJECT FILE**
- OPEN `client_interaction.pkt`.  

### **3️⃣ CHECK DEVICE CONFIGURATIONS**
- ENSURE ALL DEVICES HAVE CORRECT IP ADDRESSES.  
- SERVER → `192.168.1.100`  
- PC1 → `192.168.1.10`  
- PC2 → `192.168.1.11`  

### **4️⃣ TEST CONNECTIVITY**  
USE THE **PING COMMAND** TO CHECK CONNECTIONS:  
```bash
ping 192.168.1.100
